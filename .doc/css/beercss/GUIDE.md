# BeerCSS 요소 및 컴포넌트 사용 가이드

BeerCSS는 **Material Design 3**를 기반으로 하는 가벼운 CSS 프레임워크입니다.
HTML의 시맨틱 요소와 간단한 클래스(helpers)를 조합해 다양한 UI 컴포넌트를 제공합니다.

아래 안내는 BeerCSS 공식 문서를 참고하여 **각 요소·컴포넌트를 언제 사용하면 좋은지** 정리한 것입니다.
코딩 시 참고용으로 활용하세요.

---

## 1. 주요 요소/컴포넌트별 사용 지침

### 컴포넌트 요약 표

| 컴포넌트                      | 핵심 목적                                                                           | 적합한 상황(키워드)                                                        |
| ----------------------------- | ----------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| **Button (버튼)**             | 사용자에게 **행동을 촉구**하며 한 번의 탭으로 선택/동작을 수행                      | 기본 액션, 폼 제출, 아이콘과 함께 쓰는 동작, 주요 화면의 주요 행동(FAB 등) |
| **Badge (배지)**              | 요소의 일부분을 강조하거나 숫자/알림을 표시                                         | 메시지 수, 알림 카운트, 아이콘에 작은 숫자 표시                            |
| **Card (카드)**               | 하나의 주제에 대한 콘텐츠와 행동을 담는 표면. 계층을 명확히 하고 스캔하기 쉬워야 함 | 뉴스 카드, 제품 목록, 블로그 요약, 버튼 포함 독립 블록                     |
| **Checkbox (체크박스)**       | 여러 항목 중 복수 선택 또는 옵션을 켜고 끌 때 사용                                  | 목록 선택, 설정 토글, 동의 여부                                            |
| **Chip (칩)**                 | 입력, 속성 또는 행동을 나타내는 콤팩트 요소                                         | 태그/카테고리, 필터 토글, 작은 행동 버튼                                   |
| **Dialog (다이얼로그)**       | 작업에 대한 정보 전달, 결정 요구 또는 작은 폼을 포함한 **대화형 모달**              | 경고 메시지, 확인/취소, 간단한 폼 입력                                     |
| **Divider (구분선)**          | 리스트나 컨테이너 안의 내용 그룹을 시각적으로 구분                                  | 목록 항목 분리, 섹션 구분                                                  |
| **Expansion (확장)**          | 펼침/접기 인터페이스 제공                                                           | FAQ, 아코디언 메뉴, 다단계 목록                                            |
| **Grid (그리드)**             | 행과 열 기반으로 콘텐츠를 정렬하는 레이아웃 시스템                                  | 카드/이미지 갤러리, 반응형 레이아웃                                        |
| **Icon (아이콘)**             | 보편적인 개념을 단순한 시각 언어로 표현                                             | 버튼 보조, 네비게이션 아이템, 상태 표시                                    |
| **Input (입력 필드)**         | 사용자가 텍스트를 입력/수정                                                         | 검색창, 폼 필드, 레이블이 있는 텍스트 입력                                 |
| **List (리스트)**             | 연속적인 세로 인덱스를 제공. 각 항목은 최대 3줄까지                                 | 메뉴, 메시지 목록, 설정 목록                                               |
| **Menu (메뉴)**               | 일시적인 표면에 선택 목록을 표시                                                    | 드롭다운 옵션, 컨텍스트 메뉴, 퀵 액션                                      |
| **Navigation (네비게이션)**   | 수평 또는 수직으로 행동을 나열하는 컨테이너                                         | 상단/하단 바, 네비게이션 레일, 툴바                                        |
| **Overlay (오버레이)**        | 화면을 막아 대기 상태를 표현하는 레이어                                             | 로딩 스피너, 비동기 처리 표시                                              |
| **Page (페이지)**             | 하나의 뷰나 패널을 나타내며 여러 페이지 간 전환에 사용                              | 탭 콘텐츠, 단계별 화면                                                     |
| **Progress (프로그레스)**     | 프로세스의 진행 정도나 불특정 대기 시간을 나타냄                                    | 로딩 바, 업로드/다운로드 진행도                                            |
| **Radio (라디오 버튼)**       | 한 세트에서 **하나의 옵션만** 선택                                                  | 설문 단일 응답, 필터 단일 선택                                             |
| **Select (셀렉트)**           | 일시적 표면에서 선택 목록을 표시하는 폼 요소                                        | 긴 옵션 목록, 폼 드롭다운                                                  |
| **Shape (쉐이프)**            | M3 특유의 모양·모서리를 배경/장식에 활용                                            | 로딩 애니메이션, 그래픽 배경                                               |
| **Slider (슬라이더)**         | 값 범위에서 선택                                                                    | 볼륨/밝기 조절, 범위 값 선택                                               |
| **Snackbar (스낵바)**         | 앱 프로세스에 대한 짧은 메시지를 상단/하단에 표시, 자동 사라짐                      | 저장 완료 알림, 네트워크 상태 변화                                         |
| **Switch (스위치)**           | 단일 항목을 켜거나 끄는 토글                                                        | 다크모드, 알림 on/off 등 설정                                              |
| **Table (테이블)**            | 행·열 구조로 데이터 집합을 표시                                                     | 데이터 목록, 관리 화면, 보고서                                             |
| **Tabs (탭)**                 | 서로 다른 화면이나 데이터셋을 조직화                                                | 섹션 분리, 카테고리 전환                                                   |
| **Textarea (텍스트 영역)**    | 긴 텍스트 입력·편집 필드                                                            | 댓글 입력, 메모 작성                                                       |
| **Tooltip (툴팁)**            | 요소를 호버/포커스/터치할 때 보조 설명                                              | 아이콘 설명, 버튼 보조 안내                                                |
| **Typography (타이포그래피)** | 콘텐츠를 명확·효율적으로 전달하는 텍스트 시스템                                     | 제목, 본문, 강조 텍스트, 링크                                              |

> 💡 표는 **핵심 키워드 요약**이고, 아래 섹션에서 컴포넌트별로 좀 더 자세히 설명합니다.

## 2. 컴포넌트별 상세 가이드

### 2-1. 버튼(Button)

- **기본 버튼**

  - `button` 태그 또는 `<a class="button">` 형태로 사용.
  - 사용자에게 **명확한 행동(클릭/탭)** 을 유도할 때 사용.
  - 색상(`fill`, `primary` 등)과 크기(`small`, `medium`, `large`)로 중요도를 조절.

- **Responsive 버튼**

  - `class="button responsive"` 처럼 사용.
  - 컨테이너의 너비에 맞춰 **자동으로 크기 조절**.
  - 반응형 레이아웃에서 동일 버튼을 다양한 화면 크기에 대응시킬 때 유용.

- **FAB (Floating Action Button)**

  - 화면 주요 행동을 강조하는 **원형 버튼**.
  - 보통 화면에 **하나만** 두는 것이 좋음.
  - 예) 새 메일 작성, 새 항목 추가, 새 글 작성 등.

- **Extended FAB**

  - FAB + 텍스트 레이블.
  - 넓은 화면(웹/태블릿)에서 **주요 작업을 더 명확하게 노출**할 때 사용.

- **Button group**

  - 연관된 동작을 한 줄로 모아서 배치.
  - 예) 정렬 옵션(이름순/가격순), 페이지 이동(이전/다음).

- **Split button**
  - 메인 버튼 클릭: 기본 작업 수행
  - 옆의 작은 버튼 클릭: 관련 옵션/메뉴 표시.
  - 옵션이 몇 개 안 되고, 기본 작업이 명확할 때 사용.

---

### 2-2. 배지(Badge)

- 작은 원/사각형으로 **숫자·상태·알림** 표시.
- 아이콘, 아바타, 버튼 모서리에 붙여 사용.
- 주요 헬퍼:
  - 위치: `left`, `right`, `top`, `bottom`
  - 모양: `circle`, `square`, `round`
  - 크기: `min`(작게), 기본 크기 등
- 예시 상황
  - 메시지 개수, 장바구니에 담긴 상품 개수
  - 새 알림/새 기능 표시

---

### 2-3. 카드(Card)

- 하나의 주제(뉴스, 상품, 포스트 등)에 대한 **콘텐츠와 행동(버튼)을 묶는 표면**.
- 여러 카드를 나란히 놓으면 사용자가 **한 눈에 비교·스캔**하기 좋음.
- 구성 일반 패턴
  - 상단: 이미지/썸네일
  - 중간: 제목, 설명 텍스트
  - 하단: 행동 버튼(상세보기, 구매하기 등)
- 자주 쓰는 헬퍼
  - `border`, `round`, `no-round`
  - `padding`, `small-padding`, `medium-padding`, `large-padding`
  - `elevate`(그림자 부여)
- 사용 예
  - 뉴스 리스트, 쇼핑몰 상품 목록, 블로그 카드, 쿠폰 카드 등

---

### 2-4. 체크박스·라디오·스위치

#### Checkbox (체크박스)

- 여러 옵션 중 **복수 선택 가능**.
- `on/off` 상태를 명확하게 보여줌.
- 사용 예
  - 목록에서 여러 항목 선택
  - “이용약관 전체동의 / 일부 동의”
  - 필터에서 여러 조건 선택

#### Radio (라디오 버튼)

- 한 그룹 안에서 **하나만 선택** 가능.
- 체크박스와 헷갈리지 않게 사용해야 함.
- 사용 예
  - 설문에서 단일 응답 선택
  - 결제 수단 하나 선택
  - 정렬 기준 하나만 선택

#### Switch (스위치)

- **단일 기능을 바로 켜거나 끄는** 토글.
- 변경 즉시 저장되는 경우가 많음(확인 버튼 없이).
- 사용 예
  - 다크모드 on/off
  - 알림 on/off
  - 자동 로그인 on/off

---

### 2-5. 칩(Chip)

- 입력, 속성, 행동을 표현하는 **작고 가벼운 요소**.
- 장점: 버튼보다 작고, 태그/필터 등을 표현하기 좋음.
- 활용 패턴
  - 태그/카테고리 표시: `Vue`, `Go`, `Java` 같은 토픽 태그
  - 필터 토글: “전체 / 사용 / 미사용”
  - 선택된 값 보여주기: 선택된 필터를 상단에 칩으로 나열
- 자주 쓰는 헬퍼
  - 크기: `small`, `medium`, `large`
  - 모양: `circle`, `square`, `round`
  - 방향: `horizontal`, `vertical`
  - 테두리: `border`, `no-border`

---

### 2-6. 다이얼로그(Dialog) & 오버레이(Overlay)

#### Dialog (다이얼로그)

- 사용자에게 **중요한 결정**이나 **추가 입력**을 요구할 때.
- 기본 구조
  - 제목
  - 본문(짧은 설명/폼)
  - 하단 버튼(확인/취소 등 1~3개)
- 헬퍼
  - `small`, `medium`, `large`
  - `modal`: 배경 상호작용 차단
  - `active`: 표시/숨김 제어
- 사용 예
  - 삭제 확인
  - 간단한 설정 폼
  - 민감한 작업(결제, 권한 변경 등)

#### Overlay (오버레이)

- 전체 화면을 덮는 레이어.
- **대기 상태, 로딩, 블러 처리**에 사용.
- 헬퍼
  - 정렬: `center-align`, `middle-align` 등
  - 효과: `blur`, `small-blur`, `medium-blur`, `large-blur`
  - 상태: `active`
- 사용 예
  - 비동기 요청 대기 중 로딩 스피너
  - 모달 뒷배경 블러

---

### 2-7. 확장(Expansion) & 리스트(List)

#### Expansion (확장 패널)

- `<details>` / `<summary>` 기반의 **접이식 영역**.
- 기본 사용처
  - FAQ
  - 사이드바의 세부 메뉴
  - 옵션이 많은 폼의 고급 설정

#### List (리스트)

- 연속적인 항목을 세로로 나열.
- 각 항목은
  - 1~3줄 텍스트
  - 아이콘/아바타
  - 선택/액션 버튼 포함 가능
- 사용 예
  - 네비게이션 메뉴
  - 메시지 목록
  - 설정 리스트

---

### 2-8. 레이아웃(Layout)·그리드(Grid)·메인 레이아웃(Main layout)

#### Layout

- `absolute`, `fixed` 헬퍼로 위치 지정.
- 정렬 관련 헬퍼
  - `left-align`, `right-align`, `center-align`
  - `top-align`, `bottom-align`, `middle-align`
- 특정 요소를 화면 특정 위치에 고정하고 싶을 때 사용.

#### Grid

- **12 컬럼 기반 레이아웃**.
- 사이즈 클래스
  - `s1`~`s12` : 작은 화면
  - `m1`~`m12` : 중간 화면
  - `l1`~`l12` : 큰 화면
- 사용 팁
  - 각 칸은 반드시 `<div>`로 감싸서 사용.
  - `space`, `small-space`, `medium-space`, `large-space`로 간격 조정.
- 활용 예
  - 카드 목록/타일
  - 이미지 갤러리
  - 2~3열 레이아웃

#### Main layout

```html
<nav class="top">...</nav>
<nav class="bottom">...</nav>
<nav class="left">...</nav>
<nav class="right">...</nav>
<header class="responsive|fixed">...</header>
<main class="responsive">...</main>
<footer class="responsive|fixed">...</footer>
```

- **앱 기본 뼈대 패턴**.
- `responsive` : 최대 너비 제한 + 가운데 정렬
- `fixed` : 스크롤해도 헤더/푸터를 고정.

---

### 2-9. 네비게이션(Navigation)·탭(Tabs)·페이지(Page)

#### Navigation

- 수평/수직 네비게이션 컨테이너.
- 위치: `left`, `right`, `top`, `bottom`
- 모드:
  - `tabbed` : 탭 스타일
  - `group` : 버튼 그룹 스타일
  - `toolbar` : 툴바 스타일
- 사용 예
  - 상단/하단 네비게이션 바
  - 좌측 레일 메뉴
  - 모바일 하단 탭바

#### Tabs

- 서로 다른 콘텐츠 그룹을 빠르게 전환.
- 보통 2~6개 탭이 적당.
- 구현 패턴
  - 탭 목록: `<nav class="tabbed"> ... </nav>`
  - 탭 콘텐츠: `<div class="page">...</div>`
  - 활성 탭/페이지에 `active` 클래스 부여

#### Page

- 하나의 뷰(스크린) 단위.
- `class="page active"` 형태로 현재 페이지 표시.
- 탭/메뉴 클릭 시 JS나 `data-ui`로 활성 페이지를 변경.

---

### 2-10. 입력 계열: Input·Textarea·Select·Slider

#### Input

- 짧은 텍스트 입력에 사용.
- 헬퍼
  - `label` : 플로팅 레이블
  - `prefix` / `suffix` : 입력 앞/뒤에 아이콘/텍스트
  - `small`, `medium`, `large`, `extra` : 크기
- 사용 예
  - 로그인 폼
  - 검색창
  - 숫자/이메일 입력

#### Textarea

- 긴 텍스트 입력/편집용.
- `rows` 속성을 생략하면 내용에 따라 자동으로 늘어나는 패턴도 구현 가능.

#### Select

- 긴 옵션 목록에서 하나를 선택할 때.
- UI는 메뉴처럼 일시적으로 펼쳐졌다 접힘.
- 예) 나라 선택, 카테고리 선택, 정렬 기준 선택 등

#### Slider

- 연속적인 값 범위에서 선택.
- 사용 예
  - 볼륨/밝기 조절
  - 가격 범위 설정
  - 특정 값의 미세 조정

---

### 2-11. 테이블(Table)·타이포그래피(Typography)·미디어(Media)

#### Table

- 행/열 기반으로 데이터를 표시.
- 옵션
  - `border`, `stripes` : 구분선/줄무늬
  - 정렬: `left-align`, `center-align`, `right-align`
  - `space`, `small-space` 등으로 셀 여백 조절
- 사용 예
  - 통계 데이터
  - 관리/어드민 화면
  - 일정표

#### Typography

- 텍스트 계층을 정의하는 시스템.
- 요소
  - 제목: `<h1>` ~ `<h6>`
  - 본문: `<p>`
  - 강조: `<b>`, `<strong>`, `<em>`
- 헬퍼
  - `italic`, `bold`, `underline`, `overline`
  - `upper`, `lower`, `capitalize`
  - `small-text`, `medium-text`, `large-text`

#### Media

- 이미지/비디오/아이콘 등 미디어 요소.
- 크기: `tiny`, `small`, `medium`, `large`, `extra`
- 모양: `circle`, `round`, `no-round`, `left-round` 등
- `responsive` : 컨테이너 크기에 맞춰 자동 조절.
- 사용 예
  - 프로필 아바타
  - 카드 썸네일
  - 반응형 배너 이미지

---

### 2-12. 진행/알림: Progress·Snackbar

#### Progress

- 작업 진행 상태 또는 대기중임을 표시.
- 형태
  - 기본 바(bar)
  - `circle` : 원형 로딩
  - `wavy` : 파도 모양 애니메이션 등
- 사용 예
  - 업로드/다운로드 진행도
  - 폼 제출 중
  - 긴 연산 처리 중

#### Snackbar

- 간단한 메시지를 잠깐 보여주는 컴포넌트.
- 특징
  - 화면 상단 또는 하단에 표시
  - 4–10초 정도 후 자동 사라짐
  - 동시에 여러 개 띄우지 않는 것을 권장
- 사용 예
  - “저장되었습니다”
  - “네트워크 불안정”
  - “클립보드에 복사했습니다”

---

### 2-13. 기타: Shape·Tooltip

#### Shape

- M3 스타일의 다양한 도형/모서리.
- 헬퍼
  - 크기: `tiny`, `small`, `medium`, `large`, `extra`
  - 간격: `space`, `tiny-space`, `small-space`, `medium-space`, `large-space`
  - 애니메이션: `rotate`, `slow-rotate`, `fast-rotate`
- 사용 예
  - 로딩 애니메이션
  - 배경 장식 패턴
  - 시각적 분리용 장식 요소

#### Tooltip

- 요소 위에 마우스를 올리거나 포커스를 줄 때 나타나는 작은 설명창.
- 특징
  - 너무 긴 텍스트는 피하고 1줄 위주로.
  - 위치: `left`, `right`, `top`, `bottom`
  - 컨테이너 안에 `<div class="tooltip">...</div>` 형태로 배치.
- 사용 예
  - 아이콘만 있는 버튼 설명
  - 컬러/아이콘의 의미 설명

## 3. BeerCSS 디자인 활용 팁

1. **일관성 유지**

   - 같은 타입의 요소에는 동일한 헬퍼 조합 사용.
   - primary/secondary 색과 shape를 **역할 기준**으로 통일.

2. **반응형 우선**

   - `responsive`, Grid(s/m/l) 클래스로 모바일 → 데스크톱까지 자연스럽게 스케일.
   - 버튼/카드에도 `small/medium/large` 크기 전략을 일관되게 적용.

3. **접근성 고려**

   - Snackbar는 여러 개 겹치지 않도록.
   - Dialog는 핵심 메시지만 담고, 너무 많은 입력을 요구하지 않기.
   - 대비(contrast)와 터치 영역 최소 크기를 고려.

4. **재사용 가능한 패턴 만들기**
   - `nav + header + main + footer` 등 메인 레이아웃 패턴을 정해두고 반복.
   - 카드, 리스트 아이템, 폼 필드를 **컴포넌트 단위**로 패턴화하면 유지보수가 쉬움.
